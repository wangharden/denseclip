## P0 结论

### 1. Stage 2 主线默认 scorer

P0 结束后，主线默认 scorer 定为：

- `A1 + layer4 + neg-normal + max + patch`

原因：

- 它同时拿到本轮 `image AUROC` 最优和 `balanced` 最优。
- 3 个 seed 下波动明显小于 pixel-first 的 A2 候选。
- `max` 下 `topk_ratio` 实际不再构成有效决策项，因此这组默认配置也最简洁。

### 2. 本轮真正起主要作用的是 feature layer，不是 aggregation 小参数

P0 的主信号非常明确：

- `layer4` 明显更利于 image-level 排序。
- `local` 有时能把 pixel AUROC 拉高，但跨 seed 波动很大，不适合作为当前默认主线。
- `aggregation_mode / aggregation_stage / topk_ratio` 在大量配置上打平，属于二级因素。

这意味着后续不需要再继续扩大 scorer sweep；下一阶段应该把精力转到 prototype / retrieval / correspondence。

### 3. A1 的两种 normal scorer 没有拉开稳定差距

`one-minus-normal` 和 `neg-normal` 在 A1 上没有形成稳定可复现的性能差距。它们在 top 配置里经常给出几乎相同的排序结果，因此后续保留一个更简洁的实现即可，默认保留 `neg-normal`。

### 4. A2 的 pixel-first 候选可以保留，但不能升为默认

`A2 + normal-minus-defect + layer4` 的 pixel AUROC 的确是本轮最高，但它的问题也很明显：

- image AUROC 仍低于 A1 默认候选
- pixel 标准差达到 `0.1272`，稳定性不足

因此它更适合作为后续 P1/P2 的对照上界，而不是当前 Stage 2 默认 scorer。

### 5. blend 暂时没有继续投入的价值

`blend` 在本轮 sweep 里没有表现出独立价值，最优结果仍明显落后于主线候选，而且整体上经常贴近 `defect-minus-normal`。后续若实验预算紧张，可先将 `blend` 降级，不再作为优先扫描项。

## 对后续阶段的直接影响

### P1

主目标从“继续扫 scorer”切换为“替换单均值 prototype”：

- 优先做 retrieval / memory bank / multi-prototype
- scorer 先固定为 `A1 + layer4 + neg-normal + max + patch`
- `A2 + normal-minus-defect + layer4` 保留为 pixel-first 对照

### P2

如果 P1 仍不能同时抬高 image 和 pixel，则优先上：

- subspace residual
- lightweight correspondence / matching

而不是继续扩 scorer 组合。

### P3

重新评估 learned head 时，应直接复用 P1/P2 的最强 similarity/scorer，不再回退到旧版单均值 prototype。

### P4

扩展到 15 类前，至少先满足两点：

- 默认 scorer 已固定
- retrieval / subspace / correspondence 中至少有一条线在 `hazelnut` 上稳定优于当前 P0 主线

#### P1 结论

##### 1. P1 已完成，两轮都指向同一个结论：单均值 prototype 是主瓶颈

第一轮已经证明 `memory_bank` 明显强于 `mean` 和 `multi_prototype`；第二轮进一步证明：

- 把 `support_normal_k` 从 `8` 扩到 `16` 后，`memory_bank` 还能继续稳定抬高 image 和 pixel
- 同样的 support-size 扩展放到 `mean` 上，几乎没有带来可比增益

这说明当前性能提升并不是“单纯 support 变多了”，而是 **patch-level reference bank 的表示方式本身更有效**。

##### 2. P1 最终主线已收敛到唯一配置

P1 第二轮的 3-seed 最优配置是：

- **`A1 + layer4 + neg-normal + max + patch + memory_bank + support_normal_k=16 + reference_topk=3`**

它相对 P0 / P1 第一轮主线的提升为：

- 相对 P0 A1 default：`image 0.5857 -> 0.6732`，`pixel 0.4976 -> 0.9535`
- 相对 P1 第一轮 winner：`image 0.6472 -> 0.6732`，`pixel 0.9423 -> 0.9535`

而且波动明显收敛：

- `image std = 0.0050`
- `pixel std = 0.0071`

##### 3. 第二轮里，support-size 的有效方向很明确：`sn16` 明显优于 `sn8`，也比 `sn4` 更稳

从第二轮聚合看：

- `sn4` 的 image 可能很高，但 seed 波动大，`image std` 明显偏高
- `sn8` 已经比第一轮强，但仍落后于 `sn16`
- `sn16` 在 `refk=1/3/5` 上都进入了所有榜单顶部，说明 support 扩展在 winner family 上是有效的

因此 P1 最终不再停留在 `sn8`，而是把主线 support size 升级到 `16`。

##### 4. 第二轮里，`reference_topk` 的最优点在 `3`，但 `1/3/5` 差距不大

`refk=3` 在第二轮拿到最好的综合均值；`refk=1` 非常接近，`refk=5` 也仍然有效。  
这意味着当前主信号仍来自 `memory_bank` 本身，`kNN-k` 更像是在 winner family 内做细调，而不是决定胜负的主变量。

因此最终默认取：

- `reference_topk = 3`

同时保留一个工程判断：

- 如果后面某类数据量或显存压力特殊，`refk=1` 可以作为近似等价的简化回退

##### 5. A2 分支在 P1 不再继续扩展，P2 直接围绕 A1 最终主线展开

A2 在第一轮就没有产生可保留 winner，因此第二轮不再继续给它做 support-size 扩展。  
P1 完成后，更合理的后续顺序是：

1. 以 `A1 + memory_bank + sn16 + refk3` 作为唯一视觉主线进入 P2
2. P2 只补 `FastRef-lite / correspondence / subspace` 这类结构增强
3. A2 保留为次级历史对照，不再作为并行主线投入

##### 6. 额外 sanity check 支持把 P1 视为已完成

在 `bottle / seed42` 上，第二轮最终 winner 相对同 support-size 的 `mean` 对照也给出了极大提升：

- `mean`：`image 0.3771`，`pixel 0.2072`
- `memory_bank + sn16 + refk3`：`image 0.9568`，`pixel 0.9249`

这说明 P1 的结论并不只在 `hazelnut` 上成立。  
虽然还远不到 15 类结论，但已经足够支撑：**P1 可以正式结束，P2 直接从最终 A1 主线起步。**

#### P2 结论

##### 1. P2 已完成，但默认主线暂不切换

P2 跑完后，只有 `subspace + dim=8` 在均值上同时超过了 baseline：

- `image 0.6732 -> 0.6740`
- `pixel 0.9535 -> 0.9558`
- `pro 0.7256 -> 0.7462`

但它的 `image std` 从 `0.0050` 上升到 `0.0249`，明显恶化。  
因此按 P2 事先写定的成功标准，它**不能直接升级为新的 Stage 2 默认主线**。

##### 2. FastRef-lite 在当前口径下没有形成独立价值

FastRef-lite 在 `hazelnut / seed42` smoke 中与 baseline 几乎重合，没有带来可保留的 image、pixel 或 PRO 增益。  
因此 P2 不再继续给它补 3-seed，也不再作为当前主线候选。

##### 3. matching 没有通过 3-seed 稳定性检验

matching 在 `seed42` 上存在局部正向信号，但补齐 `43 / 44` 后：

- `match_k=1` 均值整体低于 baseline
- `match_k=3` 虽然 PRO 更高，但 image 明显回落，balanced 也更差

因此 matching 可以视为“局部可用技巧”，但还不能视为当前 P2 winner。

##### 4. subspace 是 P2 唯一应保留的结构候选，但应作为分支保留，不替换默认线

P2 真正留下来的只有：

- `A1 + memory_bank + sn16 + refk3 + subspace(dim=8)`

它的意义是：

- 证明在当前强 baseline 上，结构增强仍有可能继续抬高 pixel 和 PRO
- 也提示后续如果继续做 P3/P4，`subspace` 值得作为保留分支带入

但它当前还不满足“均值更高且稳定性不恶化”的要求，所以：

- **Stage 2 默认主线继续保持 P1 winner**
- **P2 输出一个保留候选：subspace(dim=8)**，后续再看跨类稳定性是否能把它扶正

##### 5. P2 对后续阶段的直接影响

P2 完成后，后续顺序应调整为：

1. 默认主线继续使用 `A1 + memory_bank + sn16 + refk3`
2. `subspace(dim=8)` 作为保留增强分支，优先进入后续跨类 sanity 或更大范围验证
3. `FastRef-lite` 和 `matching` 暂不继续扩大投入

#### P4 结论

##### 1. P4 已完成，当前 best full-15 visual config 是 `subspace(dim=8)`

P4 的 seed42 screening 和后续 finalist 复评都指向同一条线：

- **`A1 + memory_bank + sn16 + refk3 + subspace(dim=8)`**

这意味着 P2 留下的保留分支，在扩到 15 类后并没有消失，反而在 screening 中稳定击败了纯 baseline。

##### 2. 但这还不能写成“15 类视觉路线已经稳定”

- 类间方差仍然很大
- 弱类集中在 `leather / grid / carpet / screw`

因此更准确的说法是：

- **P4 已经判清“全 15 类该带哪条视觉线继续走”**
- **但还没有判清“这条线是否已经足够稳，可以视作通用 15 类 solved baseline”**
