# Stage 5 原因讨论提纲（给网页版 5.4 Pro）

## 背景

- 路线：`pure-visual`
- 方法族：`DINOv2 PatchCore`
- 目标：尽量对齐 `AnomalyDINO` 官方 preprocessing policy，把 full15 multi-seed aggregate `PRO` 从 repo incumbent `0.8634` 拉近论文带 `0.92+`

## 当前证据

- Stage 4 retained baseline:
  - `image_auroc = 0.974412`
  - `pixel_auroc = 0.970188`
  - `pro = 0.863387`
- Stage 5 v1 full15 official:
  - `image_auroc_mean = 0.976766`
  - `pixel_auroc_mean = 0.881120`
  - `pro_mean = 0.824271`
- 结论形态：
  - image ranking 没坏，甚至略升
  - `bottle` 和 `screw` 没有崩
  - 但 localization / `PRO` 明显变差

## 本轮改动

- 对齐方向：
  - 引入官方 8 角度 rotation
  - 把 `PRO` 改成 official-style connected-component AU-PRO
  - 把 preprocessing 当成主方法部件来对齐
- repo 内近似实现：
  - 用 patch-feature PCA foreground mask 做通用前景估计
  - 对 anomaly map 做硬 mask
  - 用 object-normalized support bank 做 reference bank 主输入

## 现有归因

- 现在最强信号不是“排序坏了”，而是“定位坏了”
- 最可疑原因不是 backbone 或 retrieval family，而是：
  - 全局、统一地施加 object-aware masking
  - query / support 几何和 mask 边界口径不完全一致
  - 仍未完全对齐的 `smaller-edge + aspect-ratio-preserving resize`
- 分类别现象也支持这个判断：
  - 对象类如 `capsule`、`screw` 有提升
  - 纹理/大面积表面类如 `leather`、`tile`、`wood`、`zipper` 的 `PRO` 下滑明显

## 想请 5.4 Pro 重点判断的问题

1. 这种“image 排名保持，但 pixel/PRO 下滑”的形态，是否最像 global object-aware masking 过度施加？
2. 在 AnomalyDINO 语境里，`masking`、`rotation`、`smaller-edge resize` 三者中，哪个最可能主导这类 localization 下滑？
3. 如果目标是先做 faithful reproduction，再做 attribution，下一步更应该：
   - 直接跑官方 repo 做外部基线
   - 还是继续在当前 repo 内把 geometry parity 补完？
4. 如果要继续留在当前 repo 内做最小修正，最优先的隔离实验应是哪一个？
   - 只改 `smaller-edge + aspect-ratio`
   - 关闭 query-side hard mask
   - 只对 object 类启用 masking
5. 从论文复现方法学上看，这条 local approximation line 是否已经接近信息增益枯竭，应该转入 “official-repo reproduction only”？
