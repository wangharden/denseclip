# 阶段四实验思考过程以及相应调整

## 1. 新分支立项 closeout

- Stage 3 已证明：在 frozen Stage 2 winner 上继续堆 `image-only residual calibrator`、`dual-head`、`frozen text prior` 都没有形成 retained winner。
- 因此本阶段另起新分支，直接测试 `defect-specific prompts + global p`，并把 `layer3/layer4` 当作视觉源对照，而不是继续修 Stage 3。

确认的判断：

- prompt 路线必须和 Stage 3 分开，不再写回 `stage3`。
- 新分支的第一轮目标不是 full-15，而是先在 `weak5_bottle / seed42` 上验证是否存在正信号。

## 2. v1 / v2 筛选后的调整

- `prompt-text v1/v2` 给出了明确正信号，尤其是 `clip_global + prompt_plus_p` 的 ranking 提升显著。
- 但 `v2` 同时暴露出两个口径问题：
  - 缺少 `ftfy` 时会静默降级。
  - 全局 pooled F1 threshold 让多个 `prompt_plus_p` 变体出现同样的 `accuracy / precision / recall / f1`，无法区分模型优劣。

因此当前阶段不应直接扩 full-15，而应先做口径修复。

## 3. reviewer 缺口说明

- 本轮流程问题不是实验思想本身，而是 orchestration 没收紧。
- 原版 `experiment-subagent-orchestrator` 只有 `implementer / runner / monitor / checker / writeback / doc-check`，缺少独立的 pre-run `reviewer`。
- 结果就是严重口径问题在跑完以后才被 checker 暴露，而不是在 official run 前被挡住。
- 当前已将 reviewer 角色补入 skill，后续训练模式默认顺序应改为：`reviewer -> implementer -> runner -> monitor -> checker -> writeback -> doc-check`。

## 4. v3 threshold-fix closeout

- `v3` 把 silent `ftfy` stub 改成显式 degraded mode。
- 同时把官方阈值协议改成了 `per-category pooled multisplit holdout best balanced accuracy`。
- 这次修复后，`weak5` 上不再出现“所有 `prompt_plus_p` 全判异常”的假阳性口径，说明修复是有效的。

确认的判断：

- 当前 prompt-text 结果应该主要看两层信息：
  - 排序层：`image_auroc_mean / image_ap_mean`
  - 分类层：`specificity / balanced_accuracy / f1`
- 单看 `accuracy` 已经不够，因为测试集正负样本严重不平衡。

## 5. full-15 closeout

- full-15 结果说明：这条路线不是无效，但也还不够强。
- `clip_global` 在 full-15 上相对 `E0` 拿到了小幅 ranking 提升。
- 但 `bottle` 回退仍在，整体分类层提升也不足以支持“阶段收口”。
- `denseclip_global` 和 `layer3_gap` 已经被 full-15 正式否掉，不再继续浪费资源。

当前收缩后的默认入口：

- 保留主线：`clip_global + defect prompts + global p`
- 保留修复点：`bottle / zipper` 的局部修复
- 放弃分支：`denseclip_global`
- 放弃分支：`layer3_gap`

下一轮不该做的事：

- 不该再回到 `layer3_gap` 主线
- 不该把当前 `full15` 小幅正增益误写成“最终 winner 已出现”
- 不该再用全局单阈值口径覆盖 category-conditioned prompt 路线

## 6. Pure-visual PCA + coreset/farthest-point closeout

- 这轮 pure-visual reviewer 先把范围收紧成一个更明确、也更容易 defend 的 concrete route：`SubspaceAD-style PCA` 负责 normal-subspace residual，`PatchCore-style greedy coreset / farthest-point` 负责 support patch memory reduction。
- 这不是任意拼接。当前 repo 里真正还没被正式验证的 pure-visual 组合，正是“`subspace` 是否能在 patch-memory 经过 coreset 去噪后形成更稳的 few-shot 路线”。
- implementer 落了三块最小代码：
  - [`fewshot/coreset.py`](/C:/Nottingham/DenseCLIP-master/fewshot/coreset.py)
  - [`fewshot/patchcore_subspace.py`](/C:/Nottingham/DenseCLIP-master/fewshot/patchcore_subspace.py)
  - [`tools/eval_structure_ablation.py`](/C:/Nottingham/DenseCLIP-master/tools/eval_structure_ablation.py) / [`tools/run_stage4_pure_visual.py`](/C:/Nottingham/DenseCLIP-master/tools/run_stage4_pure_visual.py) 的新方法入口

这轮被确认的判断：

1. `PCA + coreset/farthest-point` 在当前 repo 里是可运行、可复现、可检查的 concrete route，不存在“只讨论没落地”。
2. 这条路线的失败非常明确，不是 selector 或 seed 偶然性问题。weak5 上最好配置都没有超过 `Stage2 subspace(dim=8)`，而且 `bottle` 直接大幅塌陷。
3. 问题不只是 image ranking。多数 coreset_subspace 配置的 pixel / PRO 也同步崩掉，说明这不是“分类涨了但 localization 暂时没跟上”，而是 support patch coreset 在当前 DenseCLIP feature / support=16 的设定下破坏了正常参考空间。
4. 因为 weak5 阶段已经出现灾难性 control regression，这条路线没有资格继续推到 `full15 / seed42`，更不应占用 `seed43 / 44` 预算。

这轮推翻的判断：

1. “PatchCore 式 coreset 一定会让 few-shot support bank 更干净”在当前 repo 里不成立。至少对 `subspace` 路线来说，greedy farthest-point 更像是在 support 很少时放大了离群 patch，而不是保留了更稳的 nominal basis。
2. “只要 pure-visual 路线足够 SOTA-aligned，就值得继续和 prompt 主线并跑到 full15”也不成立。当前证据已经足以把这条 pure-visual concrete route 正式 closeout。

当前收口：

- Stage 4 不新增 pure-visual retained winner。
- Stage 4 也不把 `PCA + coreset/farthest-point` 写成下一阶段默认入口。
- 如果后续还要回到 pure-visual，新起点应更接近 `registration / semantic correspondence / stronger industrial backbone`，而不是继续在当前 `DenseCLIP layer4 + tiny support + coreset PCA` 上做局部修补。

## 7. Stage2-anchor text-hybrid closeout

- 这轮额外验证了一条更接近 `PromptAD / adapter-style conservative fusion` 的路线：冻结 `Stage 2 winner` 做 anchor，只让文本分支输出单调 residual。
- reviewer 阶段先确认了两个事实：
  - 当前 repo 真正最强的资产仍然是 `Stage 2` 的 `memory bank + subspace` 分数，而不是 prompt 分支本身。
  - 如果 hybrid 想成立，必须是“anchor 保底 + text 只做保守修正”，不能再让文本分支替代视觉 winner。

本轮被确认的判断：

1. `frozen prompt residual` 在 hybrid 里比 `global p` 更可靠。
   `plus_p` 进入 hybrid 后几乎始终退化成 `E0`，说明当前这个 post-hoc `p` 偏移并不适合作为 anchor 路线的 text adapter。
2. `Stage2-anchor` 的确能回收一部分 prompt 信号。
   `hybrid_frozen__relu_z` 在 `weak5` 上把 `image_auroc_mean` 从 `0.5947` 提到 `0.7085`，同时 `bottle / zipper` 也没有坏掉。
3. 但这条路线在当前 repo 约束下仍然不够强。
   它没有超过现有 `prompt-text incumbent (0.8118 / 0.9236)`，所以连 `weak5` screening 都没过，不能晋级 `full15`。

本轮推翻的判断：

1. “只要把 prompt 改成 anchor residual，就能自然兼顾 Stage 2 的强控制类和 Stage 4 的 prompt ranking”并不成立。
2. “tiny holdout 上做更保守的 threshold / control scaling 就足够把这条路线推进到 retained winner”也不成立。
   `midgap threshold`、`control scaling`、`support-normal normalization` 和 `support-q90 normalization` 都试过了，最强结果仍停在 `0.7085 / 0.8613`。

当前 closeout 理由：

- 从实验上看，next-best 的 conservative hybrid 变体已经做完：
  - holdout-normal z
  - support-normal z
  - support-q90 z
  - control scaling
  - boundary gating
- 从工程约束上看，真正的 `CoOp / PromptAD-like` token-level prompt learning 目前没有干净入口。
  当前实现依赖 [run_prompt_defect_text.py](/C:/Nottingham/DenseCLIP-master/run_prompt_defect_text.py) 里的 `torch.jit.load(...).encode_text(tokens)`，并没有暴露 text token embedding / transformer 路径；在这个 repo 里继续往真正的 learnable context token 走，已经不是同一条轻量 route 的局部修复。

因此当前顺序被改写为：

- 保留 Stage 4 已有 `prompt-text` 主线作为当前可用结果。
- 关闭 `Stage2-anchor text-hybrid` 路线，不再推进到 `full15 / multi-seed`。
- 如果后续还要继续文本方向，起点应该是“换 CLIP text stack / 暴露 token-level prompt learner”，而不是继续给这条 hybrid 做小修补。
