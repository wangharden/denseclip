## 1. 当前稳定主线

Stage 4 的稳定主线已经收缩为：

**frozen Stage 2 winner cache + clip_global + defect-specific prompts + global text vector `p` + multisplit holdout + per-category threshold**

解释：

- `clip_global` 是当前唯一在 `weak5` 和 `full15` 都没有被正式否掉的视觉源。
- `denseclip_global` 与 `layer3_gap` 都已经在 full-15 官方结果里输掉，不再继续。
- 分类口径必须使用 `per-category pooled multisplit holdout best balanced accuracy`，不能再回到单一 pooled F1 threshold。

## 3. 当前设计结论

已经确认：

1. 新分支不是 Stage 3 的补丁，而是一条独立主线。
2. prompt-text 路线确实能在子集上形成强正信号。
3. 口径修复后，这条路线在 full-15 上仍保留了小幅 ranking 增益。

也已经确认：

1. 当前结果还不足以替代 Stage 2 full-15 winner 作为最终项目结论。
2. 主要阻塞点已经收缩为强控制类 `bottle` 和部分强类阈值行为。
3. 继续压 `denseclip_global` 或 `layer3_gap` 没有价值。

## 4. 下一步设计

下一轮只应做 **当前 winner 的局部修复**，不再扩主线：

- 只围绕 `prompt_plus_p__clip_global`
- 目标只盯 `bottle / zipper` 回退
- 优先考虑：
  - category-aware 正则或温和 anchor
  - 更细的 defect prompt 模板清洗
  - `p` 对不同 category 的门控，而不是换视觉源

本阶段的默认成功标准：

- full-15 `image_auroc_mean` 明显高于 `E0`
- `bottle image_auroc` 不再出现当前级别回退
- 分类层 `balanced_accuracy / f1` 同时成立，而不是只有 ranking 成立
