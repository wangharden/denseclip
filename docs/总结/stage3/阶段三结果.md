- 当前协议使用 `support_defect` 作为 few-shot 训练信号，并从 `query_eval` 中排除了这些样本；因此它是 Stage 3 的 few-shot 校准协议，不应直接与标准 train-only / unsupervised MVTec 结果横向混比。

- 当前结果仍是 `single-seed / subset / screening`；它回答的是“修完 contract 后，这版实现值不值得继续扩”，而不是“整个 dual-head 方向是否彻底死亡”。

- 这次失败现在可以视为一次更公平的实现失败，而不是 score/logit contract 被破坏后的塌缩实验。

### Historical effect on next step

- 在 `P3 v2` 产出之前，这版 `P2` 仍未达到进入 `P3` 的门槛。
- 当时的默认动作不是扩 seeds，也不是回到 `P1` sweep，而是继续修 `P2` 并在同一子集同一 seed 上重跑。
- 修正优先级：
  1. 强化 image 分支的 residual calibration 与容量，而不是继续依赖当前偏弱的 pooled residual 头
  2. 重新校准 pixel 分支里 `mask BCE / Dice / consistency` 的耦合，避免训练后期继续拖垮 localization
  3. 把 `bottle` 与 weak-five 的 baseline-preserving 约束显式化，再考虑 `P3`
