## 1. P0/P1 closeout note

### 这次新增了什么

- `P0` 已经把 `weak-five + bottle / seed42` 的 Stage 2 winner cache 真正落地，并且 replay 和 Stage 2 对应子集结果对齐到 `1e-7 ~ 1e-6` 量级。
- `P1` 已经完成第一轮 `image-head-only` smoke，结果来自官方聚合目录而不是口头判断。

### 哪些早先判断被确认了

- “先做 cache，再做训练”这一步是对的。`P0` 通过后，后续不需要再怀疑 cache 本身是否引入偏差。
- “继续找 frozen visual trick 不是 Stage 3 第一优先级”这个判断没有变。`P0` 已经把当前 frozen winner 固定下来，真正该回答的是 trainable module 有没有增益。
- “`weak-five + bottle` 作为第一批 smoke 范围是合理的”这个切法也被确认了，因为它确实把 `leather / carpet` 这类关键弱类和 `bottle` 这个 strong control 同时暴露了出来。

### 哪些更细的假设被推翻了

- `P1` 没有支持“只做 image-level 全局统计校准就够”这个更弱版本的判断。  
  第一轮 `E1` 对 `bottle` 有提升，但 weak-five `image AUROC mean` 从 `0.5220` 降到 `0.5122`，主要被 `leather` 和 `carpet` 的回退拖下去。
- 这说明当前问题不是一个可以被非常小的 pooled-statistics MLP 直接吃掉的纯 image-ranking 问题。

### 这次结果如何改变后续顺序

- 保留分支：`P2 map-space dual-head`
- 降级分支：继续做 `P1 image-head-only` 的额外 sweep
- 当前不该做的事：因为 `P1` 失败就回到更大的 frozen scoring sweep；这和 Stage 2 的判断相冲突

### 为什么现在应直接转 P2

- `P1` 的负结果并不等于 “learned head 路线失败”，它只是否定了 “image-only pooled head 足够” 这一种更弱实现。
- `P1` 的 `pixel / PRO` 在这个实现里仍然是冻结 cache 指标，所以它本来就不能回答“联合训练后能不能同时保住 localization”这个核心问题。
- 真正还没被回答的是：共享 map-space stem + pixel branch + image branch 的 `dual-head`，能不能在不破坏现有 localization 的前提下把 weak classes 的 image ranking 拉起来。

### 当前保留结论

- `P0` 现在可以视为已收口，后续 Stage 3 训练直接建立在 `outputs/stage3/cache/weak5_bottle/seed42` 之上。
- `P1` 已经完成首轮 smoke，结论是“不升格、不扩 sweep、直接让位给 P2”。
- 下一步默认动作应是补齐 `MapFusionHead + mask BCE + Dice + image BCE + ranking + consistency`，而不是继续在 `ImagePoolHead` 上微调。

## 2. P2 closeout note

### 这次新增了什么

- `P2` 的第一版 `map-space dual-head` 已经真正落地并完成 `weak-five + bottle / seed42` screening。
- 这次不再是 `P1` 那种“learned image + frozen pixel”的混合评估，而是 learned image 与 learned pixel 的联合结果。

### 哪些早先判断被确认了

- `P1` 的结论是可靠的：不能因为 `image-head-only` 失败就回到 frozen scorer sweep。  
  当前真正该解决的仍然是联合 head 的实现方式，而不是回退到 Stage 2。
- `P0 cache` 继续被证明是可靠底座；这轮失败不是 cache 偏差，而是 `P2` 实现本身没有保住 baseline。

### 哪些更细的假设被推翻了

- “只要把 dual-head 按文档第一版写出来，就至少能保住 baseline 附近”这个隐含假设被推翻了。  
  这次 `E2` 不是小幅波动，而是整体同步回退，连 `bottle` 控制类都明显塌陷。
- 这说明当前瓶颈已经从“有没有必要上 dual-head”转成了“怎样写一个不会主动破坏 frozen winner 的 dual-head”。

### 这次结果如何改变后续顺序

- 保留分支：`P2 map-space dual-head` 这个大方向本身
- 降级分支：直接给当前 `P2 v1` 补 `P3 3-seed`
- 当前不该做的事：
  - 不该直接扩到 `full-15`
  - 不该回去继续做 `P1 image-head-only` sweep
  - 不该把这次负结果误写成“dual-head 路线已经失败”

### 为什么现在应先修 P2，而不是换主线

- `P2` 这次失败的性质更像实现失败，而不是路线失败：  
  image 与 pixel 同时回退，说明问题出在“如何约束 residual correction 不要破坏 frozen baseline”，而不是 `map-space dual-head` 这个方向本身。
- 当前最合理的下一步不是换新路线，而是先把 dual-head 改成更稳的实现，再在同一子集上重跑 `seed42`。

### 当前保留结论

- `P2 v1` 已经完成 screening，结论是“归档为失败实现，不晋级到 P3”。
- Stage 3 当前默认入口改为：**修正版 `P2` 重新跑 `weak-five + bottle / seed42`**。
- 下一轮优先修三个点：
  1. image 分支的 residual calibration
  2. pixel 分支与 `mask BCE / Dice / consistency` 的权重耦合
  3. `bottle` 与 weak-five 的基础保真约束

## 3. P2 v2 score-space rerun closeout note

### 这次新增了什么

- 在不改变 `P2 map-space dual-head` 主线的前提下，修掉了两个高优先级实现契约：
  - residual 路径回到 raw score space
  - `BCE / ranking` 只在 support-fitted affine calibration 后的 logit 空间优化
- 增加了 `identity_check`，现在 zero-residual 对 `E0` baseline 的 image/pixel 漂移都为 `0.0`。
- ranking 改成 smooth pairwise 形式后，正式 rerun 的 `epoch 1 rank = 0.1375`，说明它已经真实参与优化。

### 哪些早先判断被确认了

- 用户指出的 `P2 v1` 口径问题是成立的：那一版不能算干净的 dual-head 验证。
- 修掉 contract 之后，`P2 v2` 比 `v1` 明显没那么塌，说明先前相当一部分退化确实来自实现空间混淆，而不是路线本身。

### 哪些更细的假设被推翻了

- “只要修正 `score/logit contract`，`P2` 就会自然过 gate”这个更弱假设也没有成立。  
  `P2 v2` 虽然比 `v1` 更可信，但 overall 与 weak-five 仍整体低于 `E0`。
- 现在也不能再把失败归因于“ranking 分支根本没接上”，因为这轮 `rank` 在训练初期已经是非零的。

### 这次结果如何改变后续顺序

- 保留分支：`P2 map-space dual-head`
- 降级分支：继续拿 `P2 v1` 的结果解释路线本身，或者直接把当前 `P2` 送去 `P3`
- 当前不该做的事：
  - 不该回到 `P1 image-head-only` sweep
  - 不该直接扩到 `full-15`
  - 不该再用 `P2 v1` 作为当前官方 screening 依据

### 为什么现在仍应先修 P2，而不是换主线

- 现在剩下的问题已经不再是实验口径是否成立，而是模型容量、loss coupling 和 baseline-preserving constraint 是否足够稳。
- 最合理的下一步仍是沿当前这版公平 contract 继续修 `P2`，先把 `bottle` 与 weak-five 的退化压住，再决定是否值得进入 `P3`。

### 当前保留结论

- 当时官方 `P2` screening 来源改为 `outputs/stage3/p2_dual_head/weak5_bottle/seed42_scorefix`。
- `P2 v1` 降级为 historical diagnostic，不再作为 gate 结论来源。
- Stage 3 默认入口维持为：**继续修正版 `P2`，在同一子集同一 seed42 上重跑**。

## 4. P2 v2 official-source consolidation note

### 这次新增了什么

- 这次不是再开一条新路线，而是把最新完整 aggregate、identity contract 和 rank activity 一起固定成唯一官方入口。

### 哪些早先判断被确认了

- `P2 v2` 的总体判断没有变化：它已经比 `v1` 公平得多，但仍未通过 gate。
- “旧版 `P2` 要写进文档，但代码只保留最新 contract”这个整理方向是合理的。历史结果保留在文档和产物里即可，不需要继续在实现层面保留旧语义痕迹。

### 这次结果如何改变后续顺序

- 保留分支：继续修当前 `P2 v2` 实现
- 降级分支：把 `seed42_scorefix` 继续当成当前官方结论来源
- 当前不该做的事：
  - 不该再混用多个 `P2 v2` 中间目录作为 gate 依据
  - 不该在代码里继续保留已经失效的旧 `P2` 辅助逻辑

### 当前保留结论

- 当前唯一官方 `P2` screening 来源为 `outputs/stage3/p2_dual_head/weak5_bottle/seed42_score_space_v2`。
- `seed42` 继续只作为 `P2 v1` historical diagnostic；`seed42_scorefix` 继续只作为中间 rerun 轨迹。
- Stage 3 默认入口保持不变：**继续修当前 `P2`，而不是进入 `P3`**。

## 5. Objective-change note after P2

### 这次新增了什么

- 在 `P2 v2` 收口之后，任务目标被进一步收缩为：**只做图像级异常识别**。
- 这不是对 `P2` 结果的重解释，而是对 Stage 3 成功标准本身的重新定义。

### 哪些早先判断被推翻了

- “只要继续修 `P2 dual-head`，它仍是 Stage 3 默认主线”这个判断不再成立。  
  当目标还是联合 `image + pixel` 时，这个结论合理；但在目标已经缩成 image-level classification 之后，继续让主线背着 `pixel residual + mask BCE / Dice` 就变成了错目标。
- “Stage 3 仍应继续围绕 `pixel / PRO / Dice` 设 gate”这一点也被撤回。  
  这些指标对 frozen visual 结果仍有诊断价值，但不再是后续训练主线的第一判据。

### 哪些判断被保留了

- `P0` 仍是可靠底座；cache 与 frozen winner 的契约没有变化。
- `P1 / P2` 的负结果仍然有价值：它们共同说明“当前 few-shot 训练最容易做坏的，不是 frozen visual 本身，而是可学习头如何破坏 baseline”。
- “文本不该直接升成主线”这个判断被保留。文本最多只保留成 image-level 的 tie-breaker 备选。

### 这次结果如何改变后续顺序

- 新主线：`image-only residual calibrator`
- 唯一备选：`frozen text prior -> image gate`
- 降级分支：继续修当前 `dual-head + pixel loss`
- 当前不该做的事：
  - 不该继续用 `MapFusionHead` 追 `pixel / PRO`
  - 不该直接扩 `P3 3-seed` 的旧 dual-head 计划
  - 不该现在就切到 prompt tuning / adapter / LoRA

### 当前保留结论

- Stage 3 的稳定主线已经改成：**Frozen Stage 2 winner + image-only residual calibrator**。
- `P2 dual-head` 继续保留在结果文档里，作为“为什么当前不再沿 segmentation 风格训练线继续扩”的正式证据。
- 后续如果 image-only 主线接近通过但 `leather / grid` 仍不动，再开 `frozen text prior` 这条唯一备选。

## 6. P3 v2 closeout note

### 这次新增了什么

- `P3 image-only residual calibrator` 已经真正落地，并且 `py_compile` 与 monitor 都通过了。
- 这轮 `seed42_v2_rankproxy` 不再卡在 `epoch 0`，说明 selector 修复是有效的，训练过程确实在推进。

### 哪些早先判断被确认了

- `P3` 的目标必须是 image-level 校准，这一点没有变。
- 训练链路本身没有问题，monitor 全程无 alert，说明实现和落盘都是干净的。
- 但 selector 修复并不自动带来 query 提升，说明“能选到更晚 epoch”不等于“能得到更好的 retained winner”。

### 哪些更细的假设被推翻了

- “只要把选择器从 epoch 0 修掉，`P3` 就会自然过 gate”这个假设没有成立。
- 当前 holdout 侧几乎饱和，说明它更适合当诊断信号，而不是直接等价为 query 提升。

### 这次结果如何改变后续顺序

- 保留分支：`image-only residual calibrator`
- 降级分支：继续把当前 `P3 v2` 当作 retained winner
- 当前不该做的事：
  - 不该把 selector 修复误写成 smoke 通过
  - 不该把 holdout 饱和误当成 query 提升
  - 不该回退到 `dual-head + pixel loss`

### 当前保留结论

- 当前 `P3` smoke 仍未通过 gate，`P4` 继续阻塞。
- `P3 v2` 现在更准确的定位是“selector 修复成功，最终选到 `epoch 16`，但 query 增益不足”的失败样例，而不是实现错误。
- Stage 3 默认入口仍然是：**继续修 `image-only residual calibrator`**。

## 7. P3 v3 multisplit closeout note

### 这次新增了什么

- `P3 v3` 在 `seed42_v3_multisplit` 上完成了正式 smoke，`py_compile` 通过，monitor 全程 `ok`，并且训练正常结束到 `latest_epoch=71`。
- 这轮没有再卡在训练链路或合同层面，说明实现和落盘都保持干净。

### 哪些早先判断被确认了

- `P3` 仍应继续围绕 image-level calibrator，而不是回到 `dual-head + pixel loss`。
- 合同层面没有问题，`identity drift` 仍接近 `0`，`rank_smooth` 也正常，说明失败不是因为 score/logit contract 再次被破坏。
- `P3 v2` 的历史 smoke 可以保留为 selector 修复成功的证据，但它不应再代表当前官方口径。

### 哪些更细的假设被推翻了

- “只要修单次 holdout / selector 偏差，并把 gate / anchor 放松，`P3` 就会过 gate”这个假设没有成立。
- “更宽松的校准特征设计会自然带来 query 提升”也没有成立，至少在这轮 `3-resplit + hard calibration features` 上不成立。
- 这轮结果表明，问题已经不在训练是否能跑通，而在 query 侧能否真正产生可保留的增益。

### 这次结果如何改变后续顺序

- 保留分支：`image-only residual calibrator`
- 降级分支：继续把 `P3 v2` 当作历史 smoke 与诊断样例；`P3 v3` 现在也只是历史 smoke，最终官方来源已收口到 `seed42_monotonic`
- 当前不该做的事：
  - 不该把 `v3` 失败解释成合同坏掉
  - 不该把 `py_compile` / monitor 正常误写成 gate 通过
  - 不该继续回到 `dual-head + pixel loss`

### 当前保留结论

- 当前 `P3 v3` smoke 仍未通过 gate，`P4` 继续阻塞。
- `P3 v3` 的更准确定位是：selector / gate 修正后链路是干净的，但 query 侧仍然没有实质性抬升，且略差于 `P3 v2`。
- Stage 3 默认入口仍然是：**继续修 `P3 image-only`，但下一轮必须直接盯 query 增益**。

## 8. frozen text prior closeout note

### 这次新增了什么

- Stage 3 的最后备选 `frozen text prior` 也完成了 smoke，但结果与 `E0` 完全一致，`py_compile` 通过也没有改变最终判断。
- 这一轮把 Stage 3 的最终结论收口为：没有任何 retained image-level winner 优于 frozen Stage 2 winner。

### 哪些早先判断被确认了

- 训练链路本身仍然干净，`py_compile` 通过，`gate_pass_smoke=False` 只是说明策略没有带来增益，不是实现链路炸了。
- 六类 `per_category.csv` 都没有变化，说明失败不是少数坏类拖累，而是整体没有引入新的可保留信息。
- `selection_beta_mean = 0.0` 表明最优策略直接选择不加 text prior，这比“带了但效果一般”更强，属于明确的否定信号。

### 哪些更细的假设被推翻了

- “如果前面的 image-only 还差一点，那么最后加一点 text prior 就能过 gate”这个假设没有成立。
- “text prior 至少能帮到少数类，然后拖动整体收口”也没有成立，因为六类完全不变。
- 这说明 Stage 3 的问题已经不是再补一层先验，而是当前 frozen Stage 2 winner 本身就是最终可保留底座。

### 这次结果如何改变后续顺序

- 保留结论：`P3 v2`、`P3 v3`、`frozen text prior` 都只作为历史 smoke 与诊断样例。
- 降级分支：不再把任何 Stage 3 后续尝试写成“下一轮计划”。
- 当前不该做的事：
  - 不该继续为 Stage 3 新增 smoke
  - 不该把 `P4` 继续描述成阻塞等待
  - 不该把最终 winner 重新指向任何可学习 head

### 当前保留结论

- Stage 3 已收口。
- 最终 retained winner 仍是 frozen Stage 2 winner。
- 后续如果再做实验，应另起新阶段，而不是延长 Stage 3。

## 9. final diagnostic trilogy closeout note

### 这次新增了什么

- 在 Stage 3 已收口之后，又补了三件套归因诊断：`selector / objective`、`retrieval prior`、`semantic prior`。
- 这些目录只用于解释为什么当前路线不能再靠小补丁挽回，不参与新的 winner 竞争。

### 哪些早先判断被确认了

- selector 确实有偏，但只是次要症状；因为即便把口径放到 `query_best` 这个 oracle 诊断，`current / rank_only / bce_only` 也都没有拉过 `E0`。
- retrieval prior 与 semantic prior 都被搜索主动否掉，`selection_beta_mean = 0.0` 是比“效果一般”更强的负信号。
- 因此 Stage 3 的问题已经不是再调 selector、loss 配比或补一个 frozen prior 就能解决。

### 哪些更细的假设被推翻了

- “只要把 selector 再修正一点，Stage 3 就可能回正”这个假设没有成立。
- “如果 objective 改成 rank-only 或 bce-only，query 侧至少会超过 `E0`”这个假设也没有成立。
- “cached retrieval gap 或 frozen text gap 还能作为最后的小补丁”同样没有成立，因为这两个分支都被搜索直接退回 `beta = 0.0`。

### 这次结果如何改变后续顺序

- 保留结论：frozen Stage 2 winner 仍是最终 retained winner。
- 降级分支：selector、objective、retrieval prior、semantic prior 都只保留为失败归因样例。
