## 1. Stage 2 结果诊断

**已经判清的部分**只有三件事。
第一，继续做 scorer sweep 的价值很低，P0/P1/P2/P4 的收敛路径已经把视觉主线锁到了 `A1 + memory_bank + sn16 + refk3 + subspace(dim=8)`；第二，`mean prototype` 已经被正式淘汰，`memory_bank` 是稳定底座；第三，`subspace(dim=8)` 虽然在单类时不够稳，但扩到 full-15 后已经成为当前最优视觉配置。P1 正式 winner 是 `A1 + memory_bank + sn16 + refk3`，P4 则把 full-15 best visual config 明确为再叠加 `subspace(dim=8)`，同时也把弱类暴露得很清楚。 

**还没有判清的部分**也只有三件事。
第一，训练型模块到底应该学“map 到 image 的决策校准”，还是学“retrieval re-weighting / correspondence”；第二，weakest classes 到底是主要卡在表征，还是卡在局部信号到整图判别的转换；第三，旧 B 的失败到底是“head 路线无效”，还是“旧 B 的训练目标错了”。你的文档里其实已经给了一个很强的线索：旧 B 只做 image-level BCE，不用 pixel loss，不用 Dice/Focal，也没有真正利用 defect mask，这更像 supervision 设计错误，而不是 learned head 这条路线天然不行。**这是推断**，但和你当前代码描述是对齐的。

我对当前瓶颈的排序是：
**第一优先级：已经需要训练型方法继续突破，但训练对象首先应是“决策/聚合/校准 head”，不是 backbone。**
**第二优先级：弱纹理类存在表征与对应关系问题，但它不是全局首因。**
**第三优先级：retrieval/matching 还有残余空间，但已经不是 Stage 3 的第一刀。**
这不是凭空判断。我把正式结果文件 [p4_finalist_per_category.csv](../../../outputs/stage2/p4_full15_final/summary_finalist/per_category.csv) 重新看了一遍：`carpet` 和 `screw` 的 pixel 明显高于 image，更像“局部异常信号已经有了，但整图排序还没学好”；`leather` 和 `grid` 则 image、pixel 都低，更像“重复纹理下 frozen feature + 当前 normal-subspace 假设都不够”；`zipper` 更接近 region consistency/PRO 问题。**这部分是推断**，但和类别级结果是一致的。

因此，我不把现在的问题继续归因为 `scoring` 本身，也不把它主要归因为 `retrieval` 本身。Stage 2 已经证明：`memory_bank` 比 `mean` 强得多，`subspace` 也已经在 full-15 上扶正；真正没被验证的是：**当 frozen visual winner 已经固定后，一个很小的 trainable head 能不能把 weak classes 的 image-level ranking 拉起来，同时不毁掉现有 pixel localization。** 这正是你现在该切入训练的理由。 

## 2. 相关工作归纳

### A. 最贴近你当前阶段的：frozen backbone 上的小型 trainable module

**CLIP-AD** 最值得借的不是整套 zero-shot 框架，而是它对症状的判断：它明确指出 anomaly map 会出现 *opposite predictions* 和 *irrelevant highlights*，并用多层特征、feature surgery，以及额外线性层做对齐修正。对你最可迁移的点不是 prompt，而是“**小对齐层 + 多层特征 + 不把单一 anomaly map 直接硬升成整图判别**”。这和你现在 weak classes 的症状高度一致，尤其是 `carpet/screw` 这种 pixel 强于 image 的类别。单卡上现实，因为它至少说明“小模块校准”是合理方向。([arXiv][1])

**Sea-CLIP** 的可借鉴部分也不是它整套系统，而是它把 few-shot anomaly detection 写成 *feature matching + lightweight decoder*：它用 DINOv2 的 semantic-aware representation，再用一个轻量的 Anomaly Matching Decoder 把粗预测修成更精细的 mask。对你最有用的是“**decoder-style refinement head**”这件事；不适合直接照搬的部分是第二个强 encoder 和 Stable Diffusion 生成的训练异常样本，这两项都不符合你现在的单卡最小闭环。([CVF开放获取][2])

**AF-CLIP** 也支持这个判断。它不是大幅改 backbone，而是加一个 lightweight adapter，并且同时优化 class-level 和 patch-level 目标。这说明“**图像级与像素级联合训练的小模块**”是成立的；但它依赖 auxiliary datasets 上的优化，不是你现在最短路径。可借的是 joint objective，不是全套训练协议。([arXiv][3])

### B. 对你备选线最重要的：retrieval / dictionary / prototype refinement

**WinCLIP** 证明了 CLIP 在 zero-/few-normal-shot anomaly classification and segmentation 上可用，核心不是大训练，而是 dense local matching、window/patch/image-level 的有效聚合。它支持的不是 prompt 本身，而是“**few-shot anomaly signal 本来就在局部 patch comparison 里**”。这和你 Stage 2 的 memory bank 结论是同向的。([CVF开放获取][4])

**FastRecon** 和 **FastRef** 是 retrieval 备选线最相关的两篇。FastRecon 从 few normal support 重建 query 的正常版本，再做 alignment；FastRef 更进一步，把 query statistics 引入 prototype refinement，并用 optimal transport 压制 anomaly leakage，而且它是能挂到 PatchCore / FastRecon / WinCLIP 上的插件式方案。它们说明：**如果你不想先动 head，那么 next best choice 就是学一个 query-aware retrieval re-weighting。** 但你自己的 P2 smoke 已经说明 FastRef-lite 没形成独立增益，所以这条线更适合做 Stage 3 唯一备选，不适合做第一主线。([CVF开放获取][5])

**DictAS** 和 **ReMP-AD** 对你也很关键。DictAS 把 few-shot anomaly segmentation 改写成 dictionary lookup，并用 sparse lookup + query discrimination regularization 提升跨类泛化；ReMP-AD 则直接点出了 memory bank 的噪声问题，用 Intra-Class Token Retrieval 降噪，并用 Vision-Language Prior Fusion 让特征更聚焦异常区域。它们共同说明：**reference 不是“多了就行”，而是要被选择、压噪、重加权。** 这正好支撑“learned retrieval re-weighting”作为你的唯一备选线。([CVF开放获取][6])

**RegAD / category-agnostic registration** 对 `screw/zipper` 这类可能有对齐问题的类很有启发：它把 registration 当 proxy task，训练一个 category-agnostic model，对新类不再 fine-tune。它说明“先对齐再比较”确实是 FSAD 的有效范式。问题是，它意味着你要单独再建一个 proxy-training 体系，工程上已经不像“最小闭环训练”。所以它更适合当 failure-mode 解释，不适合现在先落地。([arXiv][7])

### C. 参数高效适配：是候选，但不该现在优先

**PromptAD** 是你以后真要走 prompt 路线时最值得看的一个。它只用 normal samples 做 one-class prompt learning，用 semantic concatenation 构造 anomaly prompts，再用 explicit anomaly margin 拉开 normal/anomaly prompt。它对 few-shot one-class 场景很贴近。问题在于：你现在已经有一个能出 decent pixel localization 的 frozen visual winner，最缺的不是 text semantics，而是 map-to-decision 的可学习校准。所以 PromptAD 是候选，不是第一刀。([CVF开放获取][8])

**AnomalyCLIP**、**AdaptCLIP**、**AF-CLIP** 这一类也都强，但它们的共同特征是：都更像“基于 auxiliary/base data 的全局适配”，不是“在当前 target few-shot 底座上做最小训练闭环”。AnomalyCLIP 学 object-agnostic prompts；AdaptCLIP 需要先在 base dataset 上训练三种 adapter，之后才 target-training-free；AF-CLIP 也要在 auxiliary datasets 上优化。它们更适合你后面确认“frozen map-space head 已经触顶”之后再投入。([OpenReview][9])

### D. 论文很强，但现在不值当

**Sea-CLIP 全量版** 和 **AnoGen** 都不适合你当前阶段。前者引入 DINOv2 和 SD 生成异常图；后者直接上 diffusion anomaly generation，再接 weakly supervised detector，还是建立在 DRAEM / DeSTSeg 这种另一条体系上。它们都强，但都明显超出“单卡、最小闭环、以当前 winner 为固定底座”的约束。([CVF开放获取][2])

## 3. 下一阶段训练路线判断

当前这份 Stage 3 设计需要收口到一个更明确的前提：

**如果当前目标已经收缩为“只判断整张图是否异常”，那么 Stage 3 主线不该继续围绕 pixel / PRO / segmentation 训练展开。**

基于这个前提，我的稳定判断改成下面两条。

### 主线

**唯一训练主线：Frozen Stage 2 winner 上的 image-only residual calibrator。**

固定底座不变：

* `A1 + layer4 + neg-normal + max + patch + memory_bank + support_normal_k=16 + reference_topk=3 + subspace(dim=8)`

训练模块改成一个保守的 image-level 校准器，而不是继续训练 `dual-head + pixel residual`。建议形式是：

`z_final = z_frozen + α(x) · Δ(x)`

其中：

* `z_frozen`：由 `winner_image_score` 经过固定 affine calibration 得到的 frozen logit
* `Δ(x)`：小 head 预测的 image residual
* `α(x)`：很小的 gate，初始化接近 `0`

重点不是“学一个全新的分类器”，而是**只允许模型围绕 frozen baseline 做小修正**。这比当前 `P1` 那种 fresh image prediction 更稳，也比 `P2 dual-head` 那种自由 residual 更不容易破坏强类。

### 备选

**唯一备选线：frozen text prior 作为 image-level tie-breaker。**

这里的“文本”不是 prompt tuning / adapter / LoRA，而是更保守的做法：

* 冻结 text encoder
* 手写少量 `normal / anomaly` prompts
* 计算每张图的 `text_gap`
* 只把 `text_gap` 当作额外 `1-2` 个标量特征喂给 image gate 或 residual head

也就是：文本只参与 image-level rerank，不碰 pixel map，不改 retrieval，不引入新的大训练闭环。

### 不再作为当前主线的方向

* `MapFusionHead + pixel residual + mask BCE / Dice`
* 继续围绕 `pixel AUROC / PRO / Dice` 定义 Stage 3 主 gate
* 现在就上 learnable prompt / adapter / LoRA
* 再开一轮 retrieval / scorer 大 sweep

## 4. 最小可执行训练实验计划

这里不再沿用旧的 `P2 dual-head -> P3 -> P4` 思路，而是直接按“image-only 主线先收口”的顺序重排。

### Step 0：继续复用现有 cache，不再新增 pixel 监督需求

**目标**
继续使用已经通过 `P0` 回放校验的 Stage 2 winner cache，不再为后续主线扩展新的 pixel 训练依赖。

**固定不动**

* `DenseCLIP` visual backbone
* `memory_bank`
* `subspace(dim=8)`
* 当前 `weak-five + bottle / seed42` cache

**后续主线只使用的输入**

* 5 个 cached map 的 pooled stats
* `winner_image_score`
* 可选 `knn_gap_map` 的 top-k pooled uncertainty

**不再作为主线训练输入**

* pixel GT mask
* pixel-level loss target
* PRO / localization 相关训练监督

### Step 1：image-only residual calibrator smoke

**目标**
验证 image-only 的保守残差校准，能否在不破坏 strong control 的前提下提升 weak-five 的 image ranking。

**模型**

* 输入：`pooled(map stats) + winner_image_score (+ optional text_gap)`
* 主体：先从线性 residual calibrator 开始
* 输出：单个 `image_logit`

**训练损失**

* `pairwise ranking loss`
* `image BCE`
* `anchor loss`：把高置信 normal / `bottle` 控制类拉回 frozen baseline
* 轻量 residual L2

建议第一版写得保守：

`L = 0.6 rank + 0.3 BCE + 0.2 anchor + 0.05 ||Δ||²`

**模型选择**

* 不再按 train loss 选最佳 checkpoint
* 改为按 support-holdout 的 `image AUROC / ranking proxy` 选模

**最小矩阵**

* `E0`：frozen baseline
* `E1-rescal`：image-only residual calibrator
* 范围：`weak-five + bottle`
* 先只跑 `seed42`

**验收标准**

* weak-five `image AUROC mean >= 0.55`
* 且相对 `E0` 至少提升 `+0.03`
* `bottle image AUROC` 不下降超过 `0.03`

### Step 2：image-only 主线 3-seed 复评

**目标**
判断 `image-only residual calibrator` 是否是真正可保留的训练主线，而不是 seed 偶然性。

**矩阵**

* `E0 / E1-rescal`
* seeds：`42 / 43 / 44`
* 范围仍然只保留 `weak-five + bottle`

**验收标准**

* 至少 `2/3` 个 seed 的 weak-five `image AUROC` 高于 `E0`
* 3-seed weak-five `image AUROC mean` 相对 `E0` 至少提升 `+0.03`
* `bottle image AUROC mean` 不下降超过 `0.03`

### Step 3：文本备选只做 frozen text prior 小实验

**触发条件**

只有当 `image-only residual calibrator`：

* 对 weak-five 有小幅提升
* 但仍不够稳定，或者 `leather / grid` 基本不动

时才开这一步。

**实验形式**

* 不做 prompt tuning
* 不做 adapter / LoRA
* 只新增 `text_gap` 作为 image-level 辅助特征
* 对照 `E1-rescal` 看是否能在不伤 `bottle` 的前提下继续抬高 weak-five image

**验收标准**

* weak-five `image AUROC mean` 相对 `E1-rescal` 再提升 `+0.01`
* `bottle image AUROC` 不额外下降超过 `0.01`

### Step 4：full-15 screening 与 finalist

**前提**

只有 `Step 2` 或 `Step 3` 产出了一个明确保留的 image-only winner，才扩 full-15。

**screening**

* 先跑 `seed42`
* 不开大 sweep
* 只带 `frozen baseline + retained image-only winner`

**finalist**

* 只对 screening winner 补 `seed43 / seed44`

**full-15 image-level 门槛**

* `full-15 image AUROC mean >= 0.71`
* weak-five `image AUROC mean >= 0.56`
* strong control 不出现系统性塌陷；`bottle image AUROC` 相对 frozen baseline 不下降超过 `0.03`

## 5. 路线决策建议

如果是我，下一轮最先做的 3 件事是：

1. **停止当前 `dual-head + pixel loss` 主线，不再让 Stage 3 继续围绕 segmentation 目标迭代。**
2. **重做一个显式带 frozen anchor 的 image-only residual calibrator，并把 checkpoint 选择改成 holdout image metric。**
3. **只有当 image-only 主线接近通过时，才加 frozen text prior 作为唯一备选。**

我推荐的**唯一训练主线**是：

**Frozen `A1 + memory_bank + sn16 + refk3 + subspace(dim=8)` + image-only residual calibrator**

我保留的**唯一备选线**是：

**Frozen text prior 进入 image gate / tie-breaker**

我明确**不推荐现在投入**的方向是：

**继续修当前 `MapFusionHead` 的 pixel 分支、重新把 Stage 3 gate 绑回 `pixel / PRO`、以及现在就上 prompt tuning / adapter / LoRA。**

最后一句话收口：

**既然当前目标已经缩成“图像级异常识别”，Stage 3 就不该继续训练一个同时背着 segmentation 目标的模型；它应先验证 frozen winner 是否能被一个保守的 image-level calibrator 学得更稳。**

## 6. Stage 3 closeout

后续写回已经覆盖 `P3 v2`、`P3 v3` 和 `frozen text prior` 三轮最终 smoke。结论已经收口为：

- Stage 3 没有产出一个优于 frozen Stage 2 winner 的 retained image-level winner。
- `P3 v3` 说明 selector / gate 修复能让训练链路保持干净，但 query 侧仍无实质增益。
- `frozen text prior` 作为最后备选也没有带来任何类别级变化，最优搜索直接退回“不加 text prior”。
- 因此 Stage 3 已闭环，最终 retained winner 仍是 frozen Stage 2 winner。
- 后续不再继续 `P4`，如果要继续研究，只能另起新阶段，而不是延长当前 Stage 3 链路。
